# patterns (模式仓库)

> **酒店运营的模式与反模式** — 什么管用，什么不管用。

从实践中提炼的可复用模式。

## 已知模式

### 模式1: 三层分层
```
治(即时应) → 析(归因) → 预(趋势)
```
GSM投诉管理验证了这种分层的有效性。

### 模式2: 4D决策模型
```
收益层 → 理性层 → 情绪层 → 底线层
```
赔偿决策的金字塔模型。

### 模式3: 9+N体系
```
核心站点(9) + 扩展项目(N)
```
核心稳定的同时保持扩展性。

## 反模式

- **一次解决问题** — 治标不治本，投诉会复发
- **数据孤岛** — 各站不关联，看不到全貌
- **过度设计** — 还没跑就想着优化
