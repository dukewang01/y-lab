# F&B Graph Builder (FB知识图谱基建)

> 所有餐饮分析的底座。图谱稳了，上层分析才能跑。

FB站是九站体系中最复杂的一站——6,661个实体、6,547条关系。这个技能负责FB知识图谱的搭建、维护、对齐、修复。

## 脚本 (7个)

| 脚本 | 功能 |
|:-----|:-----|
| `_import_fb_module.py` | FB模块通用导入器 |
| `_restructure_fb.py` | FB图谱结构重建 (7.0KB) |
| `_show_fb_structure.py` | 显示FB图谱完整结构 |
| `_show_full_fb.py` | 全量FB数据展示 (5.7KB) |
| `_check_bq_import.py` | BQ(宴会)数据导入检查 |
| `_fix_missing_types.py` | 修复缺失的实体类型 |
| `_fix_cases.py` | 修复案例数据 (11.8KB) |

## 7+2营业点对齐

FB图谱的核心对齐工作——确保7个堂食店 + 2个零售点的数据一致：

| 类型 | 营业点 | 状态 |
|:----|:-------|:----:|
| 🍽️ 堂食 | OPEN / YUXI / BACIO / BEER / YUAN / 宴会 / 送餐 | ✅ |
| 🛒 零售 | 美食屋 / 市集 | ✅ |

## 关键对齐步骤

1. 产品→营业点归属 (BELONGS_TO 1,885条)
2. 产品→CRM偏好标签 (RECOMMENDED_FOR 818条)
3. 营业点→FAQ索引 (FAQ桥接 358条)
4. FIN站↔FB站营收数据双向验证
