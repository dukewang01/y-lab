# y-mep — 物业工程机电管理系统

> **Mechanical, Electrical & Plumbing** — 酒店设备维保日历 + 故障预警 + 全生命周期管理

---

## 背景

传统酒店物业工程管理普遍处于"设备坏了才修"的被动模式：

- ❌ 电梯困人 → 投诉 + 赔偿
- ❌ 空调夏季停机 → 客人拒住 → 营收损失
- ❌ 锅炉/水泵突发故障 → 紧急检修费用翻倍
- ❌ 设备台账散落 Excel，到期不知、过期不管

**y-mep** 将设备管理从"救火模式"升级为"日历驱动 + 预警前置 + 全生命周期追踪"。

---

## 定位

面向 **酒店/商业综合体/物业集团** 工程技术部门的轻量级机电管理工具：

```
┌──────────────────────────────────────────────────┐
│                 y-mep 机电引擎                      │
├──────────┬──────────┬──────────┬──────────────────┤
│ schedule │  alarm   │ lifecycle │      cost        │
│ 保养日历  │ 故障预警  │ 全生命周期  │    维修成本       │
├──────────┴──────────┴──────────┴──────────────────┤
│              集成层: 飞书 / 钉钉 / 短信              │
└──────────────────────────────────────────────────┘
```

---

## 架构模块

| 模块 | 职责 | 核心能力 |
|------|------|----------|
| **schedule** | 维保日历 | 按设备类型/品牌/使用年月自动生成保养计划；日历视图；到期推送 |
| **alarm** | 故障预警 | 基于工况指标的异常检测；分级告警（提醒/预警/紧急）；SLA 时效追踪 |
| **lifecycle** | 全生命周期 | 从入库→使用→维保→大修→报废的全链路档案；决策何时修 Vs 换 |
| **cost** | 维修成本 | 单台累计维保成本 vs 设备残值分析；供应商报价对比；成本超支预警 |

## 使用场景

### 场景 1：每日巡检
```python
from mep.schedule import DailyRoutine

engineer = DailyRoutine("2026-07-01")
engineer.check("LIFT-001")    # → "正常，下次保养 2026-07-15"
engineer.check("PUMP-004")    # → "⚠️ 已过期 15 天，请立即安排保养"
```

### 场景 2：告警推送
```python
from mep.alarm import AlarmEngine

alarm = AlarmEngine()
alarm.evaluate(hvac_temp=35.2)        # → WarningLevel.MEDIUM
alarm.dispatch(["飞书群", "短信"])     # 自动推送
```

### 场景 3：生命周期决策
```python
from mep.lifecycle import LifecycleManager

lc = LifecycleManager("HVAC-002")
lc.years_in_service()    # → 8.3
lc.repair_vs_replace()   # → "建议更换（累计维保已超残值 120%）"
```

## 处理流程

```
设备入库
    │
    ▼
初始化生命周期档案 ───→ 设置首次维保日期
    │
    ▼
每日/每周保养日历 ───→ 到期检测 ───→ 推送提醒
    │                       │
    │                   [已完成]     [未处理]
    │                       │           │
    │                   记录日志       升级预警
    │                                   │
    ▼                              紧急维修
设备状态汇总 ────→ 成本核算 ────→ 换新决策
    │
    ▼
报废出库 ────→ 归档
```
