# complaint-classifier — 投诉自动分类器

**Status: Planning** | **Target: v0.1**

把GSM投诉自动归因到菜品/服务/环境，追踪趋势变化。

## 要解决的问题

> "这周投诉主要是什么？哪类问题在恶化？"

酒店每个月几十上百条投诉，人工归类费时且不统一。自动分类+趋势追踪，让管理层一眼看到问题全貌。

## 分类体系

```
投诉
├── 菜品（品质/口味/温度/异物/份量）
├── 服务（效率/态度/沟通/失误）
├── 设施（空调/电梯/水电/网络）
├── 环境（噪音/清洁/氛围）
└── 其他（定价/政策/预定）
```

## 方法路线

| 阶段 | 方法 | 输出 |
|------|------|------|
| v0.1 | 关键词规则引擎 | 自动分类 + 置信度 |
| v0.2 | + TF-IDF + 分类器 | 更细粒度分类 |
| v0.3 | + 趋势预警 | "XX类问题连续3周上升" |

## 用法

```python
from complaint_classifier import classify

result = classify("菜太咸了，服务员态度也很差")
# {"category": "菜品", "sub_category": "口味", "confidence": 0.92}
```
